Redis

Hosting Redis on Fly is nice. node-resque is great for background jobs. Redcon is nice Redis based server.

Notes​

  • Redis is great for shared memory across service instances. Solves the 'how can I make every instance of this app aware of something' problem. Examples: Session store backed by redis. Cache layer backed by redis. Async atomic Job queue.
